# 부서진 학교 개발 체크리스트

이 문서는 PRD를 기준으로 실제 개발 중 하나씩 체크하면서 진행하기 위한 작업 목록이다.

체크 규칙:

- `[ ]` 아직 안 함
- `[x]` 완료
- 작업을 끝냈다고 느껴도 반드시 바로 아래 `완료 기준`까지 확인한 뒤 체크한다.

## 0. 개발 전 결정

- [x] 목표 지점을 정한다.
  - 완료 기준: `탈출문`으로 선택했다.
- [x] 괴물에게 잡혔을 때 처리를 정한다.
  - 완료 기준: 1차 버전은 `시작 지점으로 이동`으로 확정했다.
- [x] 첫 테스트 인원을 정한다.
  - 완료 기준: 1차 테스트 인원을 `1명`, 가족 테스트 인원을 `2~6명`으로 정했다.
- [x] 공포 분위기 수위를 정한다.
  - 완료 기준: 어린이가 플레이 가능한 `살짝 무섭고 귀여운 스타일`로 정했다.

## 1. Roblox 프로젝트 준비

- [x] Roblox Studio에서 새 프로젝트를 만든다.
  - 완료 기준: Rojo 프로젝트와 `BrokenSchool.rbxlx` 파일을 만들었다.
- [x] 프로젝트 이름을 `부서진 학교`로 정한다.
  - 완료 기준: 게임 이름 또는 내부 프로젝트 이름에서 `부서진 학교`를 확인할 수 있다.
- [ ] 기본 테스트 실행을 해본다.
  - 완료 기준: Play 버튼을 눌렀을 때 플레이어 캐릭터가 정상 생성된다.
- [x] 작업 저장 방식을 정한다.
  - 완료 기준: Rojo 소스 구조와 `.rbxlx` 빌드 산출물로 저장 방식을 정했다.

## 2. 기본 맵 제작

  - 완료 기준: 플레이어가 시작할 수 있는 교실 공간이 있다.
- [x] 복도를 만든다.
  - 완료 기준: 시작 교실에서 다른 공간으로 이동 가능한 복도가 있다.
- [x] 교실을 2개 이상 만든다.
  - 완료 기준: 플레이어가 들어갈 수 있는 교실이 최소 2개 있다.
- [x] 꺾이는 통로 또는 계단을 만든다.
  - 완료 기준: 맵이 일직선이 아니라 숨거나 돌아갈 수 있는 구조가 있다.
- [x] 목표 지점까지 이어지는 길을 만든다.
  - 완료 기준: 시작 위치에서 목표 지점까지 걸어서 도착할 수 있다.
- [x] 괴물 순찰 구역을 만든다.
  - 완료 기준: 괴물이 움직일 복도 또는 구역이 따로 정해져 있다.
- [ ] 맵 충돌을 확인한다.
  - 완료 기준: 플레이어가 벽을 뚫거나 바닥 아래로 떨어지지 않는다.

## 3. 분위기와 장식

- [x] 밤 분위기를 만든다.
  - 완료 기준: 조명 또는 하늘 설정이 낮이 아니라 밤처럼 보인다.

  - 완료 기준: 너무 어둡지 않지만 긴장감 있는 밝기다.
- [x] 부서진 책상 또는 의자를 배치한다.
  - 완료 기준: `부서진 학교` 느낌이 나는 오브젝트가 최소 3개 있다.
- [x] 금 간 벽 또는 어지러운 장식을 추가한다.
  - 완료 기준: 빈 학교가 아니라 망가진 장소처럼 보인다.
- [x] 주요 길은 알아볼 수 있게 둔다.
  - 완료 기준: 장식 때문에 이동 경로가 막히거나 헷갈리지 않는다.

## 4. 시작 위치와 목표 지점

- [x] 플레이어 시작 위치를 배치한다.
  - 완료 기준: 게임 시작 시 플레이어가 시작 교실에서 생성된다.
- [x] 목표 지점 파트를 배치한다.
  - 완료 기준: 정해진 목표 지점에 눈에 보이는 탈출 영역이 있다.
- [x] 목표 지점에 닿는지 감지한다.
  - 완료 기준: 플레이어가 목표 지점에 닿았을 때 스크립트가 실행된다.
- [x] 승리 처리를 구현한다.
  - 완료 기준: 목표 지점에 닿으면 `탈출 성공!` 메시지가 나온다.
- [x] 승리 후 위치 처리를 구현한다.
  - 완료 기준: 승리한 플레이어가 승리 위치로 이동하거나 더 이상 방해받지 않는다.

- [x] 어린이에게 과하지 않은 무서움인지 확인한다.

- [x] 괴물 모델에 충돌 기준 파트를 정한다.
  - 완료 기준: 플레이어와 닿았는지 판단할 파트가 있다.
- [x] 괴물의 시작 위치를 정한다.
  - 완료 기준: 괴물이 시작 지점 바로 앞이 아니라 순찰 구역에서 시작한다.

## 6. 눈괴물 순찰

- [x] 괴물 순찰 포인트를 2개 이상 만든다.
  - 완료 기준: 괴물이 왕복하거나 순서대로 이동할 위치가 있다.
- [x] 괴물이 순찰 포인트로 이동하게 한다.
  - 완료 기준: 게임 실행 시 괴물이 멈춰 있지 않고 이동한다.
- [x] 순찰 속도를 조정한다.
  - 완료 기준: 플레이어가 도망칠 수 있지만 방심하면 잡힐 정도다.
- [ ] 괴물이 벽에 끼지 않는지 확인한다.
  - 완료 기준: 1분 이상 실행해도 괴물이 맵에 끼지 않는다.

## 7. 눈괴물 추격과 잡힘 처리

- [x] 괴물의 감지 범위를 정한다.
  - 완료 기준: 플레이어가 가까이 오면 괴물이 반응할 거리 값이 정해져 있다.
- [x] 플레이어가 가까우면 추격하게 한다.
  - 완료 기준: 감지 범위 안에 들어가면 괴물이 플레이어 쪽으로 이동한다.
- [x] 플레이어가 멀어지면 순찰로 돌아가게 한다.
  - 완료 기준: 추격이 끝나면 괴물이 다시 순찰한다.
- [x] 괴물 접촉 감지를 구현한다.
  - 완료 기준: 괴물이 플레이어에게 닿았을 때 잡힘 처리가 실행된다.
- [x] 잡히면 시작 지점으로 이동시킨다.
  - 완료 기준: 플레이어가 잡힌 뒤 시작 위치에서 다시 플레이할 수 있다.
- [x] 잡힘 메시지를 표시한다.
  - 완료 기준: 잡혔을 때 `괴물에게 잡혔어요!` 메시지가 보인다.

  - 완료 기준: 괴물에게 잡히면 화면이 짧게 붉게 번쩍인다.
- [x] 위험 경고 효과를 추가한다.
  - 완료 기준: 괴물이 가까워지면 화면에 위험 경고가 표시된다.

## 11. 1인 테스트

- [ ] 게임 시작 테스트를 한다.
  - 완료 기준: 플레이어가 시작 교실에서 정상 생성된다.
- [ ] 목표 지점까지 이동 테스트를 한다.
  - 완료 기준: 시작 위치에서 목표 지점까지 막힘 없이 갈 수 있다.
- [ ] 승리 테스트를 한다.
  - 완료 기준: 목표 지점에 닿으면 승리 메시지가 나온다.
- [ ] 괴물 순찰 테스트를 한다.
  - 완료 기준: 괴물이 정해진 구역을 계속 움직인다.
- [ ] 괴물 추격 테스트를 한다.
  - 완료 기준: 가까이 가면 괴물이 따라온다.
- [ ] 잡힘 테스트를 한다.
  - 완료 기준: 괴물에게 닿으면 시작 지점으로 돌아간다.
- [ ] 슬라이딩 테스트를 한다.
  - 완료 기준: 슬라이딩이 동작하고 쿨타임도 적용된다.
- [ ] 퓨즈 수집 테스트를 한다.
  - 완료 기준: 퓨즈를 먹을 때마다 진행도가 올라간다.
- [ ] 발전기 복구 테스트를 한다.
  - 완료 기준: 퓨즈 3개 수집 후 발전기에 닿으면 탈출문이 열린다.
- [ ] 손전등 테스트를 한다.
  - 완료 기준: 손전등을 먹고 F키로 켜고 끌 수 있다.
- [ ] 보호 부적 테스트를 한다.
  - 완료 기준: 보호 부적이 있을 때 첫 잡힘이 방어된다.
- [ ] 사물함 은신 테스트를 한다.
  - 완료 기준: 사물함 안에서는 괴물이 추격하지 않는다.
- [ ] 깨진 유리 함정 테스트를 한다.
  - 완료 기준: 밟으면 안내 메시지가 나오고 괴물이 더 멀리서 반응한다.
- [ ] Pathfinding 괴물 이동 테스트를 한다.
  - 완료 기준: 괴물이 벽에 오래 끼지 않고 순찰/추격을 계속한다.
- [ ] 손전등 눈부심 테스트를 한다.
  - 완료 기준: 손전등을 괴물 쪽으로 비추면 괴물 속도가 느려지고 안내 메시지가 나온다.
- [ ] 잡힘 화면 효과 테스트를 한다.
  - 완료 기준: 잡혔을 때 화면이 빨갛게 번쩍인다.
- [ ] 위험 경고 테스트를 한다.
  - 완료 기준: 괴물이 가까워지면 위험 경고가 표시된다.
- [ ] 최종 추격 조명 테스트를 한다.
  - 완료 기준: 발전기 복구 후 붉은 조명과 화면 연출이 보인다.
- [ ] 도서관 귀신 테스트를 한다.
  - 완료 기준: 귀신이 떠다니고 가까이 가면 차가운 기운 메시지가 나온다.
- [ ] 대형 맵 이동 테스트를 한다.
  - 완료 기준: 시작점에서 북쪽/남쪽 별관, 체육관, 운동장, 발전기실, 탈출문까지 이동할 수 있다.
- [ ] 전체 플레이 3회 테스트를 한다.
  - 완료 기준: 3번 연속으로 시작부터 승리 또는 잡힘까지 진행할 수 있다.
