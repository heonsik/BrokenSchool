# 부서진 학교 PRD

## 1. 개요

`부서진 학교`는 밤의 학교에서 괴물을 피해 목표 지점까지 도착하는 Roblox 생존 탈출 게임이다. 플레이어는 어두운 학교 안을 돌아다니며 위험을 피하고, 슬라이딩을 활용해 괴물의 추격에서 벗어나 탈출해야 한다.

첫 버전은 작고 완성 가능한 범위로 만든다. 핵심은 "밤의 학교", "움직이는 괴물", "목표 지점 도착", "슬라이딩"이다.

## 2. 목표

- 자녀가 설계한 아이디어를 실제 Roblox 게임으로 구현한다.
- 짧은 시간 안에 플레이 가능한 첫 버전을 만든다.
- 이후 아이템, 여러 괴물, 라운드 시스템을 추가할 수 있는 구조로 만든다.

## 3. 대상 플레이어

- Roblox를 즐기는 어린이와 가족
- 숨바꼭질, 생존, 탈출 게임을 좋아하는 플레이어
- 짧고 이해하기 쉬운 목표가 있는 게임을 선호하는 플레이어

## 4. 게임 콘셉트

### 게임 이름

부서진 학교

### 장르

숨바꼭질 / 생존 / 탈출

### 배경

밤의 학교

### 기본 이야기

플레이어들은 밤의 부서진 학교에 갇혀 있다. 학교 안에는 입이 찢어진 눈괴물이 돌아다니고, 플레이어는 괴물을 피해 목표 지점까지 도착해야 한다.

### 핵심 재미

- 어두운 학교를 탐험하는 긴장감
- 괴물을 피해 도망치는 생존감
- 슬라이딩으로 위기를 빠져나가는 조작 재미
- 친구들과 함께 탈출하는 협동감

## 5. 플레이 방식

### 플레이어 수

- 설계 목표: 최대 30명
- 1차 개발 목표: 1명에서 6명까지 안정적으로 플레이
- 확장 목표: 성능 확인 후 12명, 20명, 30명 순서로 확장

### 난이도

보통

### 승리 조건

목표 지점에 도착하면 승리한다.

1차 버전에서는 학교의 `탈출문` 또는 `운동장 출구`에 도착하면 승리한다.

### 실패 조건

1차 버전에서는 완전한 게임 오버 대신, 괴물에게 잡히면 시작 지점으로 돌아간다.

추후 버전에서는 목숨, 제한 시간, 관전 모드를 추가할 수 있다.

## 6. 핵심 기능

### 6.1 학교 맵

1차 버전 맵 구성:

- 시작 교실
- 복도
- 교실 2개 이상
- 계단 또는 꺾이는 통로
- 목표 지점
- 괴물 순찰 구역

분위기:

- 밤 느낌
- 일부 조명만 켜져 있음
- 부서진 책상, 떨어진 의자, 금 간 벽 등으로 "부서진 학교" 느낌 표현

### 6.2 플레이어 이동

기본 이동:

- 걷기
- 점프
- 달리기 또는 기본 Roblox 이동 사용

특별 이동:

- 슬라이딩

슬라이딩 규칙:

- 키 입력 시 짧은 거리 빠르게 미끄러진다.
- 재사용 대기시간이 있다.
- 괴물을 피하거나 낮은 장애물을 통과하는 데 사용할 수 있다.

1차 버전에서는 슬라이딩을 "짧은 대시"처럼 구현해도 된다.

### 6.3 괴물

몬스터 이름:

- 눈괴물

외형:

- 큰 눈
- 찢어진 입
- 무섭지만 어린이가 보기에는 너무 과하지 않은 스타일

1차 버전 행동:

- 정해진 길을 순찰한다.
- 플레이어가 가까이 오면 쫓아온다.
- 플레이어와 닿으면 플레이어를 시작 지점으로 돌려보낸다.

추후 행동:

- 소리를 듣고 이동
- 손전등 빛을 보면 멈춤
- 특정 구역에서는 더 빠르게 이동

### 6.4 목표 지점

1차 후보:

- 학교 정문
- 운동장 출구
- 비상문

목표 지점에 도착하면:

- "탈출 성공!" 메시지를 보여준다.
- 플레이어를 승리 위치로 이동시킨다.
- 라운드 시스템이 생기기 전까지는 다시 시작 버튼 또는 자동 리스폰을 제공한다.

## 7. 아이템

1차 버전 필수 아이템은 없다. 먼저 이동, 목표, 괴물부터 완성한다.

추가 후보:

- 손전등: 어두운 곳을 밝힌다.
- 열쇠: 잠긴 문을 연다.
- 에너지 음료: 잠깐 빠르게 이동한다.
- 치료 아이템: 추후 체력 시스템이 생기면 사용한다.
- 투명 망토: 짧은 시간 괴물에게 보이지 않는다.

## 8. 화면과 UI

1차 버전 UI:

- 목표 안내: "출구를 찾아 탈출하세요"
- 잡혔을 때 메시지: "괴물에게 잡혔어요!"
- 승리 메시지: "탈출 성공!"
- 슬라이딩 쿨타임 표시

추후 UI:

- 남은 시간
- 생존자 수
- 현재 목표
- 획득한 아이템

## 9. 사운드

1차 버전:

- 배경 분위기 소리
- 괴물이 가까워질 때 효과음
- 잡혔을 때 효과음
- 탈출 성공 효과음

주의:

- 어린이가 플레이하므로 너무 큰 비명 소리나 과한 공포 연출은 피한다.

## 10. 첫 개발 범위

### MVP 목표

처음 만들 버전에서는 아래 기능만 완성한다.

- 밤의 학교 맵
- 시작 위치
- 목표 지점
- 목표 지점 도착 시 승리 처리
- 눈괴물 1마리
- 눈괴물 순찰
- 눈괴물 접촉 시 시작 지점으로 이동
- 슬라이딩 기능
- 기본 안내 메시지

### MVP에서 제외

- 최대 30명 최적화
- 여러 종류의 괴물
- 복잡한 아이템 시스템
- 라운드 매칭 시스템
- 상점
- 점수 랭킹
- 저장 데이터

## 11. 개발 순서

1. Roblox 프로젝트 생성
2. 기본 학교 맵 제작
3. 플레이어 시작 지점과 목표 지점 배치
4. 목표 지점 도착 감지 구현
5. 괴물 모델 배치
6. 괴물 순찰 이동 구현
7. 괴물 접촉 시 리스폰 구현
8. 슬라이딩 구현
9. UI 메시지 추가
10. 가족 테스트 후 난이도 조정

## 12. 확장 아이디어

- 열쇠를 찾아야 출구가 열리는 모드
- 여러 개의 출구 중 하나만 진짜인 모드
- 괴물이 점점 빨라지는 모드
- 친구를 구출하는 협동 목표
- 교실마다 다른 퍼즐
- 숨을 수 있는 사물함
- 손전등 배터리 시스템
- 라운드 시작 전 역할 선택
- 플레이어 중 한 명이 괴물이 되는 모드

## 13. 성공 기준

1차 버전 성공 기준:

- 플레이어가 게임에 들어와 시작 지점에서 출발할 수 있다.
- 목표 지점까지 이동하면 승리 메시지가 나온다.
- 괴물이 움직인다.
- 괴물에게 닿으면 시작 지점으로 돌아간다.
- 슬라이딩을 사용할 수 있다.
- 자녀가 직접 플레이하며 "내가 만든 게임 같다"고 느낄 수 있다.

## 14. 확인할 질문

개발 전에 정하면 좋은 질문:

- 목표 지점은 정문, 운동장, 비상문 중 어디인가?
- 괴물은 몇 마리까지 늘리고 싶은가?
- 잡히면 바로 탈락인가, 시작점으로 돌아가는가?
- 손전등과 열쇠 중 어떤 아이템을 먼저 넣을까?
- 분위기는 무섭게 할까, 귀엽고 살짝 긴장되는 느낌으로 할까?
